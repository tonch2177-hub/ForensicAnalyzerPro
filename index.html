How PINs Work

Each generated PIN is 4 digits with a unique Session ID (e.g. MCK-001).

All PIN validation is handled server-side. The scanner sends the PIN to /api/validate-pin, the server validates it, and sends a Discord webhook notification.

When the dashboard server is running, PINs are persisted in data/pins.json. Scanners call POST /api/validate-pin with the PIN and the server responds.

Dashboard Server

1. Run npm install && npm start from the dashboard/ folder.

2. The server starts on http://localhost:3000.

3. Configure dashboard/config.json with your Discord webhook URL for notifications.

{ "apiUrl": "http://localhost:3000", "discordWebhook": "https://discord.com/api/webhooks/..." }

4. Configure the WPF scanner's config.json with the same apiUrl.

5. PINs are stored in dashboard/data/pins.json, scans in dashboard/data/scans.json.
